What is a Claude Skill?

A packaged module of instructions, heuristics, and examples that teach Claude how to perform a specific review task reliably.

  • Mission: What you want Claude to achieve when the skill runs.
  • Heuristics: Guardrails Claude should apply while reasoning.
  • Examples: Representative scenarios anchoring expected behaviour.

Why export via Awesome Reviewers?

The same teams who curate the reviewer library produce first-class Claude Skill exports. Keep your AI pair-reviewers, CLI tooling, and automation in sync without rewriting guidance for each surface.

  • Export individual reviewers as skills.
  • Generate entire categories in a single run.
  • Compose dependency-aware super-skills for your stack.

Awesome Reviewers CLI

Convert the library into Claude Skills in minutes

Our tools/awesome2claude.py utility clones the repo, parses every reviewer prompt, and emits Claude-compatible SKILL.md files.

  • Supports filters for single reviewers or full categories.
  • Understands dependency graphs for combined skills.
  • Outputs ready-to-use bundles for Claude Workflows.
pip install click pyyaml
python tools/awesome2claude.py --output-dir ./claude_skills --overwrite
python tools/awesome2claude.py --project-dir ~/code/my-service --combined-skill-slug project-dependencies

Run python tools/awesome2claude.py --help for the complete option list.

Consistent AI interfaces

Enforce predictable SDK ergonomics across providers.

  • Prefer .response over .rawResponse for parity.
  • Group provider-specific options under shared providerOptions.
  • Ensure docs use real model IDs and supported configuration.

Async error callbacks

Surface background failures through explicit hooks.

  • Propagate errors into logging rather than failing silently.
  • Expose callback parameters when async work continues after return.
  • Protect streaming consumers from mid-flight aborts.

Document configuration decisions

Make build and dependency choices auditable.

  • Pin dependency versions when compatibility spans packages.
  • Organise dependencies by publish target.
  • Include tests validating changes to build targets.
